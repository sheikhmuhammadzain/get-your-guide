# Smart Trip AI — Project Context

> **Read this file first in every conversation to understand the codebase.**

## What is this app?

AI-powered Turkey travel planner — personalized itineraries, AI chat assistant, interactive map, tour marketplace, and realtime travel data. Built as a university final-year project.

## Tech Stack

- **Framework**: Next.js 16 (App Router, Turbopack)
- **Language**: TypeScript (strict)
- **Styling**: Tailwind CSS v4 + PostCSS
- **Database**: MongoDB Atlas via Mongoose 9
- **Auth**: NextAuth v4 (credentials — email/password + bcrypt)
- **AI/LLM**: OpenRouter SDK + OpenAI SDK (hybrid, deterministic fallback)
- **Maps**: Google Maps JavaScript API
- **Validation**: Zod v4
- **Icons**: Lucide React
- **Runtime/Package Manager**: Bun
- **Testing**: Vitest
- **CI**: GitHub Actions

## Architecture

```
app/                    → Next.js App Router (pages + API routes)
  (app)/                → Authenticated routes (dashboard, admin, planner, itineraries, etc.)
  api/v1/               → Versioned REST API (all endpoints under /api/v1/*)
  auth/                 → Login/Register pages
  products/, cart/,     → Commerce pages
  checkout/, wishlist/
  about/, blog/, etc.   → Static/marketing pages

components/             → Shared React components
  Header.tsx            → Global nav (auth, cart, wishlist, theme, language/currency)
  ItineraryGenerator.tsx → Main AI planner (632 lines — destinations, budget, dates, realtime)
  AiAssistant.tsx       → Floating AI chat widget (SSE streaming, 470 lines)
  TurkeyMap.tsx         → Interactive Google Maps (route builder, geolocation, 389 lines)
  ProductCard.tsx       → Tour card component
  LanguageCurrencyDialog.tsx → Language & currency preferences
  CurrencyAmount.tsx    → Dynamic currency display

modules/                → Domain business logic (clean architecture)
  ai/                   → Chat agent (intent detection), chat service (streaming), OpenRouter/OpenAI clients
  itineraries/          → Planner service (deterministic), itinerary CRUD, Mongoose model
  attractions/          → Turkey attractions catalog (model, repo, service)
  realtime/             → Weather (OpenWeatherMap), Currency (ExchangeRate-API), Transport (Google Distance Matrix), cache
  users/                → User management + preferences
  orders/               → Order processing
  products/             → Product options
  feedback/             → User feedback
  wishlist/             → Wishlist management
  commerce/             → Cart logic
  admin/                → Admin analytics
  auth/                 → NextAuth config + options
  shared/               → Zod schemas, pagination, RFC 7807 errors, response helpers

lib/                    → Infrastructure
  db/                   → MongoDB + Mongoose connection singletons
  auth/                 → getAuthSession() helper
  data.ts               → Static product catalog (12 Turkey tours — Cappadocia, Istanbul, Ephesus, etc.)
  preferences-client.ts → useAppPreferences() hook (syncs header ↔ planner)
  env/                  → Server environment variable loader
  observability/        → Logger
  rate-limit/           → API rate limiting

types/                  → Shared TS types (ItineraryRequest, GeneratedItinerary, BudgetLevel, etc.)
theme/                  → Design tokens (brand colors)
scripts/                → Seed scripts (attractions, admin user)
openapi/                → OpenAPI spec (travel-planner-v1.yaml)
```

## Module Pattern

Every domain module follows:
```
module/
├── *.model.ts          → Mongoose schema & model
├── *.repository.ts     → Data access layer (MongoDB queries)
└── *.service.ts        → Business logic (called by API routes)
```

## Key Design Decisions

1. **Hybrid AI**: OpenRouter → OpenAI fallback → deterministic algorithm fallback
2. **Agentic chat**: AI detects intent (recommendation/booking/general) and enriches with product data
3. **SSE streaming**: Chat responses stream via Server-Sent Events
4. **Client-side prefs**: Language/currency sync via useAppPreferences() hook + custom events
5. **Static products**: 12 tours in lib/data.ts (not DB) — suits the fixed catalog
6. **Dark mode**: localStorage + suppressHydrationWarning
7. **API conventions**: Versioned (/api/v1/*), RFC 7807 errors, Zod validation, cursor pagination
8. **SEO**: Full OpenGraph, Twitter Cards, JSON-LD structured data, canonical URLs

## API Endpoints

- `GET /api/v1/health` — Health check
- `GET|POST /api/v1/itineraries` — List / create itineraries
- `POST /api/v1/itineraries/generate` — AI-generate itinerary
- `GET|PATCH|DELETE /api/v1/itineraries/:id` — Single itinerary
- `POST /api/v1/assistant/chat` — AI chat (SSE stream)
- `GET /api/v1/attractions` — Turkey attractions
- `GET /api/v1/realtime/weather|currency|transport` — Live data
- `GET|PATCH /api/v1/users/me/preferences` — User preferences
- `GET /api/v1/users/me` — Current user profile
- `GET /api/v1/users/me/dashboard` — Dashboard data
- `POST /api/v1/feedback` — Submit feedback
- `GET /api/v1/orders` — Order history
- `POST /api/v1/checkout` — Checkout
- `GET|POST|DELETE /api/v1/wishlist` — Wishlist
- `GET /api/v1/products` — Product listings
- `GET /api/v1/admin/overview|users|orders|itineraries` — Admin endpoints

## Environment Variables

- `MONGODB_URI` — MongoDB connection string
- `NEXTAUTH_SECRET` / `NEXTAUTH_URL` — Auth config
- `ADMIN_EMAILS` — Comma-separated admin emails
- `OPENROUTER_API_KEY` / `OPENROUTER_MODEL` — Primary LLM
- `OPENAI_API_KEY` / `OPENAI_MODEL` — Fallback LLM
- `OPENWEATHER_API_KEY` — Weather data
- `EXCHANGERATE_API_KEY` — Currency rates
- `GOOGLE_MAPS_API_KEY` — Maps + Distance Matrix
